# 데이터베이스 무료 티어 비교

## 📊 3가지 옵션 비교

### 1. SQLite (로컬 파일) ⭐ 가장 무료

**비용:**
- ✅ **완전 무료** (제한 없음)
- 파일 기반, 서버에 저장

**제한사항:**
- ❌ Vercel 서버리스 환경에서 사용 불가 (파일 시스템이 영구적이지 않음)
- ✅ 로컬 개발, Railway, Render 같은 단일 인스턴스 서버에서는 완벽

**용량:**
- 무제한 (서버 디스크 용량까지)

**추천:**
- 로컬 개발용
- 단일 인스턴스 서버 배포 (Railway, Render 등)

---

### 2. Turso (LibSQL) ⭐ Vercel 추천, 무료 티어 우수

**비용:**
- ✅ **무료 티어:**
  - 데이터베이스: 500개
  - 스토리지: 1GB
  - 읽기: 500만 건/월
  - 쓰기: 100만 건/월
  - 지역: 1개

**장점:**
- ✅ SQLite와 100% 호환 (기존 코드 그대로 사용)
- ✅ Vercel 공식 추천
- ✅ 서버리스 환경에서 완벽 작동
- ✅ 무료 티어가 충분히 넉넉함 (소규모 프로젝트에 적합)

**제한사항:**
- 무료 티어는 1GB 스토리지
- 월 500만 읽기, 100만 쓰기 제한

**추천:**
- ✅ **Vercel 배포 시 가장 추천**
- 소규모~중규모 프로젝트

---

### 3. Vercel Postgres (PostgreSQL)

**비용:**
- ✅ **무료 티어:**
  - 스토리지: 256MB
  - 시간당 60초 제한 (Hobby 플랜)
  - 일일 백업: 7일

**장점:**
- ✅ Vercel과 완전 통합
- ✅ 자동 백업

**제한사항:**
- ❌ 무료 티어 스토리지가 작음 (256MB)
- ❌ 시간당 60초 제한 (트래픽이 많으면 제한)
- ❌ 코드 변경 필요 (PostgreSQL 문법)

**추천:**
- 대용량 데이터가 필요한 경우
- PostgreSQL 특화 기능이 필요한 경우

## 🎯 추천 순위

### Vercel 배포 시:
1. **🥇 Turso** (가장 추천)
   - 무료 티어가 넉넉함 (1GB)
   - SQLite 호환으로 코드 변경 최소화
   - Vercel 공식 추천

2. **🥈 Vercel Postgres**
   - 스토리지가 작지만 (256MB) 간단한 프로젝트에는 충분
   - Vercel과 완전 통합

3. **🥉 SQLite**
   - Vercel에서는 사용 불가

### 단일 서버 배포 시 (Railway, Render 등):
1. **🥇 SQLite** (가장 추천)
   - 완전 무료, 제한 없음
   - 설정 간단

## ✅ 최종 추천

**Vercel 배포 시: Turso 사용**

이유:
1. ✅ 무료 티어가 충분히 넉넉함
2. ✅ SQLite 호환 (코드 변경 최소)
3. ✅ Vercel 공식 추천
4. ✅ 서버리스 환경에서 완벽 작동

**환경 변수 설정:**
```env
TURSO_DATABASE_URL=libsql://your-database.turso.io
TURSO_AUTH_TOKEN=your-auth-token
```
